# 安全政策

RertChat 非常重视安全问题。以下是我们处理安全问题的政策。

## 报告安全问题

如果你发现了安全漏洞或问题，请立即联系我们：

### 报告方式

1. **GitHub Security Advisories**（推荐）
   - 前往 [Security Advisories](https://github.com/RuanMingze/RertChat-Ai/security/advisories/new)
   - 使用 "Report a vulnerability" 选项

2. **电子邮件**
   - 发送邮件至项目维护者

### 报告内容

请包含以下信息：

- 安全问题的详细描述
- 复现步骤
- 潜在影响的评估
- 建议的修复方案（如有）

### 响应时间

- 我们会在 **24 小时内** 确认收到你的报告
- 我们会在 **7 天内** 提供初步评估
- 修复后我们会通知你并公开致谢

## API Key 安全

### 如果你的 API Key 泄露

**立即采取以下行动：**

1. **前往 Keys 页面轮转 Key**
   - 访问 <https://rertx.dpdns.org/keys>
   - 找到泄露的 Key，点击"轮转"按钮
   - 新 Key 会自动生成并复制，旧 Key 立即失效

2. **检查使用记录**
   - 查看 API 调用日志，确认是否有异常使用

3. **更新应用配置**
   - 将新 Key 更新到所有使用该 Key 的应用中

### 保护 API Key 的最佳实践

- **永不**将 API Key 提交到 Git 等版本控制系统
- **永不**在前端代码中硬编码 API Key
- 使用**环境变量**存储 API Key
- 定期**轮转** API Key
- 如果 Key 泄露，**立即轮转**

## 滥用政策

### 禁止行为

以下行为将导致服务被禁用：

1. **未经授权的使用**
   - 使用他人账户的 API Key
   - 尝试访问不属于你的资源

2. **恶意用途**
   - 用于恶意软件或网络攻击
   - 用于垃圾邮件或钓鱼
   - 用于任何非法目的

3. **过度使用**
   - 超出合理范围的请求频率
   - 试图绕过速率限制

4. **资源滥用**
   - 使用服务进行加密货币挖矿
   - 使用服务进行大规模爬虫
   - 使用服务进行暴力破解

### 惩罚措施

| 违规级别 | 行为 | 惩罚 |
|---------|------|------|
| 轻微 | 首次轻微违规 | 警告通知 |
| 中等 | 重复违规或过度使用 | 临时禁用服务 |
| 严重 | 恶意用途或安全危害 | 永久禁用并追究法律责任 |

### 申诉流程

如果你认为你的账户被错误禁用：

1. 通过 GitHub Issues 联系维护者
2. 提供账户信息和使用说明
3. 我们会在 48 小时内审核

## 漏洞赏金

我们会考虑为重要的安全漏洞报告提供赏金，具体金额根据漏洞影响程度决定。

## 致谢

我们感谢所有帮助我们提升安全性的安全研究者和用户。
